Module 1 — Fundamentals

OFDM —
Why 5G uses multicarrier

Before understanding PSS correlation or CORESET tables, you need to understand why 5G uses OFDM at all. This section starts from the fundamental problem that multicarrier transmission solves — multipath propagation — and builds up to the complete OFDM signal model used in NR.

TS 38.211 §5.3 TS 38.211 §4.2

The fundamental problem — multipath

In wireless communication, a transmitted signal does not travel along a single direct path. It reflects off buildings, the ground, and other objects. Multiple copies of the same signal arrive at the receiver at slightly different times. This is called multipath propagation.

The time difference between the first arriving copy and the last is called the delay spread. In a typical urban environment, delay spread is 1–5 μs. In large cells it can reach 10–20 μs.

Multipath propagation — multiple copies arrive at different times Physical channel model

When a single carrier waveform is transmitted, delayed copies of it arrive and overlap with the current symbol, corrupting it. This is called Inter-Symbol Interference (ISI). The faster you transmit (higher data rate → shorter symbols), the worse ISI becomes because each symbol is shorter than the delay spread.

The traditional fix — an equalizer — works well for narrow channels but becomes extremely complex for the wide bandwidths (100 MHz+) that 5G requires.

The OFDM solution

Instead of one wideband carrier, OFDM splits the bandwidth into hundreds of narrow subcarriers, each carrying a low-rate signal. A subcarrier narrow enough to have a symbol duration much longer than the delay spread experiences flat fading (not frequency-selective), which a single tap equalizer can handle trivially.

For 5G NR with 30 kHz subcarrier spacing: symbol duration = 33.33 μs — roughly 10–30× longer than typical urban delay spread. Each subcarrier sees a nearly constant channel across its symbol.

For downlink and uplink transmission, CP-OFDM (cyclic prefix OFDM) shall be used. For uplink transmission, DFT-s-OFDM may additionally be used for single-layer transmission.

Orthogonality — why subcarriers don't interfere

With hundreds of subcarriers occupying overlapping spectrum, why don't they interfere with each other? The key is orthogonality. Two sinusoids at frequencies f₁ and f₂ are orthogonal if their inner product over one symbol period T is zero:

Orthogonality conditionTS 38.211 §4.2
// Integral of product of two subcarriers over symbol period T:
(1/T) × ∫₀ᵀ e^(j2π·k·Δf·t) × e^(−j2π·m·Δf·t) dt

= (1/T) × ∫₀ᵀ e^(j2π·(k−m)·Δf·t) dt

= { 1   if k = m  (same subcarrier)
  { 0   if k ≠ m  (different subcarriers) ✓

// This holds ONLY when subcarrier spacing Δf = 1/T
// For 30 kHz SCS: T = 1/30000 = 33.33 μs  ← exactly one symbol period

This is why subcarrier spacing must equal exactly 1/T. In 5G NR with 30 kHz SCS, the symbol useful duration is exactly 33.33 μs = 1/30,000 s. The subcarriers are spaced 30 kHz apart — the minimum spacing that preserves orthogonality.

In the frequency domain, each subcarrier's spectrum is a sinc function. Its nulls fall exactly on all other subcarrier center frequencies. At the receiver FFT, each bin captures only energy from its own subcarrier.

The OFDM signal — transmitter

At the transmitter, data symbols (QPSK, 16-QAM, 64-QAM, 256-QAM, or 1024-QAM) are mapped to frequency-domain subcarriers. The time-domain waveform is generated by an Inverse Fast Fourier Transform (IFFT).

Cyclic prefix — solving the remaining multipath problem

Even though each subcarrier's symbol is long, multipath still causes the end of one symbol to overlap with the start of the next at the receiver. This ISI between adjacent OFDM symbols destroys the orthogonality we just established.

The solution is the cyclic prefix (CP): copy the last N_CP samples of each OFDM symbol and prepend them as a guard interval. At the receiver, the CP is discarded. If the CP is longer than the channel delay spread, no ISI occurs.

The time-continuous signal sl,μ(t) on antenna port p in OFDM symbol l in one subframe is defined as:

s(t) = Σₖ aₖ · e^(j2π(k+½)Δf(t−N_u·κ·Tc))    −N_CP·κ·Tc ≤ t < N_u·κ·Tc

where N_CP is the cyclic prefix length and N_u is the FFT size.

OFDM numbers for 30 kHz SCS — our example

For band n78 with 100 MHz channel bandwidth and 30 kHz subcarrier spacing:

// Subcarrier spacing and symbol duration
Δf           = 30,000 Hz
T_useful     = 1/Δf = 33.333 μs   (2048 samples at 61.44 Msps)

// Cyclic prefix (normal CP, TS 38.211 Table 5.3.1-1)

// Total symbol duration

// Active subcarriers in 100 MHz channel
N_active     = 3276 subcarriers  (273 RBs × 12)
Total BW     = 3276 × 30 kHz = 98.28 MHz  (guard bands on edges)

// FFT size
N_FFT        = 4096  (next power of 2 above 3276)
Sampling rate = N_FFT × Δf = 4096 × 30000 = 122.88 Msps

OFDM receiver — the FFT

At the receiver, after removing the cyclic prefix, an FFT converts the time-domain samples back to frequency-domain symbols. For a channel with flat response on each subcarrier, the received symbol on subcarrier k is:

OFDM reception and equalization
// After FFT, received symbol on subcarrier k:
Y[k] = H[k] × X[k] + N[k]

// Where:
H[k]  = complex channel gain on subcarrier k  (estimated via DMRS)
X[k]  = transmitted data symbol
N[k]  = noise

// One-tap equalization (trivially simple):
X̂[k] = Y[k] / H[k]
      = X[k] + N[k]/H[k]

// Compare to single-carrier: requires Nᵢ-tap FIR equalizer
// where Nᵢ = delay_spread / sample_period
// For 100 MHz BW: Nᵢ ≈ 5μs × 100MHz = 500 taps → complex!
// OFDM: always 1 tap regardless of bandwidth → elegant.
