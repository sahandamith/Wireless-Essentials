Module 1 — Fundamentals

Numerology &
Frame Structure

5G NR does not use a single fixed subcarrier spacing. It defines five numerologies — μ = 0 through 4 — each doubling the SCS of the previous. This section explains what numerology is, why different values exist, and how the 10 ms radio frame is divided into slots, symbols, and subframes.

TS 38.211 §4.2 TS 38.211 §4.3

What is numerology?

In 5G NR, numerology refers to the subcarrier spacing (SCS) and its consequences — symbol duration, CP length, slot duration, and number of slots per subframe. All of these scale together based on a single parameter: μ (mu).

For the supported transmission numerologies, the subcarrier spacing and cyclic prefix are given in Table 4.2-1. The supported subcarrier spacings are Δf = 2^μ · 15 kHz where μ ∈ {0, 1, 2, 3, 4}.
3GPP TS 38.211, Section 4.2

Each increase in μ doubles the subcarrier spacing, which halves the symbol duration. This scales all time-domain quantities together while keeping the FFT structure consistent.

Why different numerologies?

No single SCS is optimal for all scenarios. The choice involves a trade-off between two competing requirements:

Large cells (sub-6 GHz, wide coverage): Long delay spreads (up to 10–20 μs) require a long CP, which means long symbols (low SCS, μ=0 or 1). Phase noise is low at these frequencies, so narrow subcarriers are fine.

mmWave (FR2, small cells): Delay spreads are short (0.1–1 μs) but phase noise is severe. Phase noise rotates subcarriers and causes inter-carrier interference — the fix is wider subcarriers (higher SCS, μ=3 or 4). Wider subcarriers also mean shorter symbols, which reduces latency.

Radio frame structure

The 5G NR time axis is organized into a hierarchy of fixed-duration units. The radio frame is the top level, always 10 ms, regardless of numerology.

Downlink and uplink transmissions are organized into radio frames with a duration of T_f = 10 ms. Each radio frame consists of 10 subframes with a duration of T_sf = 1 ms. For numerology μ, there are 2^μ slots in each subframe, each containing 14 OFDM symbols with normal cyclic prefix.

SFN — System Frame Number

Radio frames are numbered sequentially using the System Frame Number (SFN). SFN counts from 0 to 1023, then wraps back to 0. The full cycle is 1024 × 10 ms = 10.24 seconds.

The SFN is critical for system timing. Scheduling, paging occasions, measurement gaps, SIB transmissions — all are defined relative to SFN and slot number. When a UE decodes the MIB, reconstructing the full SFN is one of the first things it does.

Slot structure — 14 symbols

Every slot contains 14 OFDM symbols (for normal CP). Symbols within a slot are numbered 0–13. Symbol 0 has a slightly longer CP (160 samples vs 144) to maintain slot timing alignment across subframes.
