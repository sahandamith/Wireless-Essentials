Module 1 — Fundamentals

Resource Grid —
The time-frequency canvas

Everything transmitted in 5G NR — data, control, reference signals — is mapped onto a two-dimensional grid of time and frequency. Understanding this resource grid is essential before reading any scheduling, CORESET, or PDSCH documentation. This section defines every unit from Resource Element up to Bandwidth Part.

TS 38.211 §4.4 TS 38.211 §4.5 TS 38.213 §12

Resource Element — the smallest unit

A Resource Element (RE) is defined as one subcarrier in one OFDM symbol. It is the smallest unit of the resource grid. Each RE carries one complex modulation symbol — BPSK, QPSK, 16-QAM, 64-QAM, 256-QAM, or 1024-QAM.

For 30 kHz SCS, one RE spans 30 kHz in frequency and 33.33 μs in time (the useful symbol duration, not including CP).

One resource element (RE) is the basic unit of the resource grid, defined as one OFDM subcarrier during one OFDM symbol. The resource grid is defined for a given numerology μ and antenna port p as a set of resource elements indexed by (k, l) where k is the subcarrier index and l is the OFDM symbol index within a subframe.
3GPP TS 38.211, Section 4.4.1

Resource Block — the scheduling unit

A Resource Block (RB) is defined as 12 consecutive subcarriers in the frequency domain. It spans exactly one slot in time (14 symbols). RBs are the granularity at which the scheduler assigns frequency resources to UEs.

For 30 kHz SCS, one RB = 12 × 30 kHz = 360 kHz in bandwidth.

Channel bandwidth and RB counts

Not all subcarriers in the channel bandwidth carry data. Guard bands at the edges prevent interference with adjacent channels. The number of active RBs depends on channel bandwidth and SCS.

Point A — the absolute frequency reference

All RB positions in 5G NR are defined relative to a reference point called Point A. Point A is the center of subcarrier 0 of RB 0. It is not necessarily the lowest edge of the channel — it is an abstract reference from which all resource positions are computed.

The MIB field ssb-SubcarrierOffset (k_SSB) defines the offset between Point A and the SSB's lowest subcarrier. This allows the UE to calculate Point A from the SSB position, then compute all other resource positions from Point A.

Point A is the reference point for resource block grid and is located at the centre of the subcarrier 0 of the resource block 0 in the common resource block grid. The common resource block numbering starts at subcarrier 0 of common resource block 0 at Point A.
3GPP TS 38.211, Section 4.4.4.1

CRB, PRB, VRB — three flavours of resource block

3GPP defines three types of resource blocks that can be confusing:

RB types — CRB vs PRB vs VRB TS 38.211 §4.4.4
TypeFull nameNumbering referenceUsed for
CRB Common Resource Block Absolute, from Point A System-level positioning (CORESET#0, BWP)
PRB Physical Resource Block Relative to BWP start PDSCH/PUSCH resource assignment within a BWP
VRB Virtual Resource Block Same numbering as PRB Interleaved mapping — VRBs map to non-consecutive PRBs for frequency diversity
Practical rule: DCI resource assignments are in PRBs (relative to BWP start). CORESET and BWP positions are in CRBs (relative to Point A). When computing absolute frequencies, always convert back through Point A.

Bandwidth Parts (BWP)

A Bandwidth Part is a contiguous set of physical resource blocks configured for a specific UE. A UE does not need to receive the full 100 MHz channel at all times. Instead, it can be configured with a narrower BWP, saving power during periods of low activity.

Up to 4 DL BWPs and 4 UL BWPs can be configured per UE. Only one DL and one UL BWP are active at any time. BWP switching is triggered by DCI or inactivity timer.

For a UE configured with bandwidth part bwp-Id, the active bandwidth part is defined by startRB and numRBs within the common resource block grid. The UE shall not expect the active bandwidth part to exceed the UE channel bandwidth capability.
3GPP TS 38.213, Section 12
